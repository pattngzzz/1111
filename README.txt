==========================================
       销售预测系统 - 使用说明
==========================================

📋 项目概述
-----------
这是一个基于机器学习的销售预测Web应用，使用LightGBM算法
能够分析历史销售数据，预测未来销量趋势，并提供详细的可视化分析。

🚀 快速开始
----------

### 💻 方法1：命令行运行
1. 打开命令提示符或PowerShell
2. 进入项目目录：`cd e:\a销售预测`
3. 安装依赖：`pip install -r requirements.txt`
4. 运行应用：`streamlit run app.py`
5. 在浏览器中访问：`http://localhost:8501`

### 🐍 方法2：PyCharm运行（推荐）
1. 打开PyCharm项目
2. 在右上角运行配置中选择 "streamlit run app"
   - 如果没有这个配置：右键点击 `app.py` → "Run 'streamlit run app'"
3. 或者手动配置：
   - 点击"Run"菜单 → "Edit Configurations"
   - 添加新配置：
     - **Name**: `streamlit run app`
     - **Script path**: `streamlit`
     - **Parameters**: `run app.py`
     - **Working directory**: `E:\a销售预测`
4. 点击运行按钮

📊 功能说明
-----------

1. 数据上传
   - 支持CSV格式的销售数据文件
   - 推荐使用示例数据：Dataset_train.csv
   - 数据必须包含：T1列（销量）、F10列（日期）

2. 预测设置
   - 可调整预测天数：1-30天
   - 默认预测未来7天

3. 结果分析
   - 模型评估指标：MAE、RMSE、MAPE
   - 销量趋势图表
   - 特征重要性分析
   - 残差诊断统计

4. 数据导出
   - 支持下载预测结果为CSV文件
   - 便于后续分析和处理

📈 数据格式要求
--------------

✅ 项目支持多种CSV格式，具有智能列名检测和自动重命名功能：

必需列（自动检测）：
- 销量相关：T1, 销量, 销售量, sales, amount, value, quantity
- 日期相关：F10, 日期, date, 时间, time, timestamp

可选列：
- F1-F9: 其他特征变量（数值型）
- F7, F8: 布尔型特征（支持Y/N, Yes/No, True/False, 1/0等格式）

✅ 支持的日期格式：
- YYYY-MM-DD (2023-01-01)
- YYYY/M/D (2023/1/1)  
- M/D/YYYY (01/03/2023)
- YYYY.MM.DD (2023.01.04)

✅ 数据质量要求：
- 销量数据应为正数（非正数将被过滤）
- 日期格式应该能够解析
- 支持缺失值自动填充
- 自动过滤无效数据

📋 支持的CSV格式示例：
示例1（标准格式）：
T1,F1,F10
100,1,2023-01-01

示例2（中文列名）：
销售量,特征1,日期
100,1,2023-01-01

示例3（英文列名）：
sales,feature1,date
100,1,2023-01-01

🔧 技术架构
-----------

前端框架：Streamlit
机器学习：LightGBM
数据处理：Pandas + NumPy
可视化：Plotly
统计分析：SciPy + Statsmodels

📁 项目文件结构
--------------
app.py              - 主应用文件
utils.py            - 核心功能模块（包含格式兼容性）
Dataset_train.csv   - 示例数据
requirements.txt    - 依赖包列表
README.txt          - 使用说明（本文件）
.idea/              - PyCharm项目配置
.venv/              - Python虚拟环境

⚠️ 注意事项
-----------

1. 首次运行前请确保安装所有依赖包：
   pip install -r requirements.txt

2. 数据质量要求：
   - 确保日期格式正确
   - 销量数据应为正数
   - 建议至少包含100条历史记录

3. 性能建议：
   - 数据量越大，预测效果越好
   - 建议使用至少1年的历史数据
   - 预测天数不宜过长（建议1-30天）

🛠️ 故障排除
-----------

问题1：模块导入错误
解决：运行 pip install -r requirements.txt

问题2：数据读取错误
解决：检查CSV文件格式和列名

问题3：模型训练失败
解决：检查数据质量和缺失值

问题4：预测结果异常
解决：确保有足够的历史数据
